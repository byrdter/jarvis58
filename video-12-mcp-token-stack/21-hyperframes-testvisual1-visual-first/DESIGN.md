# Video 12 TestVisual1 Visual-First Overlay

## Style Prompt

Build the middle section as a fast, technical, editorial animation: dense but legible system diagrams, glowing context windows, token meters, search beams, perspective tool libraries, and kinetic data cards. The visuals should feel like a designed technical documentary, not a slide deck.

## Colors

- Canvas: `#020611`, `#06111f`, `#091727`
- Flow cyan: `#34f5ff`
- Optimized green: `#59ffad`
- Warning amber: `#ffb23f`
- Failure red: `#ff4164`
- Evidence blue: `#3da5ff`
- Text: `#f7fbff`, `#a9c4d8`

## Typography

- Interface and headings: Inter/system sans.
- Code-like cards: SFMono-Regular/ui-monospace.
- Use bold condensed-looking weights through high font weight, uppercase, and tight but non-negative tracking.

## Motion Rules

- Every subsegment must have a system action: flow, scan, compression, expansion, benchmark climb, or redistribution.
- Use the one-second pauses as scene-reset moments.
- Let visuals lead the VO by roughly half a second when possible.
- Keep key text still for at least 1.5 seconds after each build.
- Vary entrances: panels slide, cards arc, bands snap, beams sweep, meters compress, layers redistribute.

## What Not To Do

- Do not show production labels like "Segment 004" or "Video 12".
- Do not use plain dark boxes with static text as the main visual.
- Do not place entering text over readable existing text.
- Do not use fades as the only motion.
