# Video 12 Two-Scene Proof

This proof deliberately avoids the "panel with text" habit. Each scene must feel like a directed screen event: objects enter, systems react, counters run, cursors make choices, data moves, and visual focus changes with the narration.

## Visual Language

- Dark cinematic technical canvas, but not a flat dashboard.
- Large typography only for the few words the viewer should remember.
- Layered objects: tool tiles, rays, meters, source windows, terminal panes, cursor, warning stamps, and animated counters.
- Keep text sparse. The narration carries the paragraph; the screen carries the action.
- Use saturated accent color changes: cyan for working flow, red for overload, green for successful CLI, amber for caution.

## What Not To Do

- Do not make every scene a card grid.
- Do not use cursor movement unless it clicks or points to something meaningful.
- Do not leave the same visual state on screen for the whole segment.
- Do not place text over text.
- Do not use still images as final art; build layered objects directly.
