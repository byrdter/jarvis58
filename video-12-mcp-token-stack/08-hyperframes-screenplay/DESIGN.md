# Video 12 Screenplay Rebuild Design

## Style Prompt

Build Video 12 as a directed technical explainer, not a slideshow and not a generic VS Code demo. Each segment must feel like its own visual scene while still belonging to one premium cyber-technical system. Use dark glass surfaces, large readable type, animated source cards, glowing data streams, token meters, architecture layers, comparison boards, and carefully cropped HeyGen avatar layers.

## Colors

- Canvas: `#060b14`
- Deep panel: `#0d1728`
- Glass panel: `rgba(18, 35, 58, 0.78)`
- Cyan action: `#3ff7ff`
- Blue source: `#5da7ff`
- Green success: `#72f6a2`
- Amber warning: `#ffbe55`
- Red overload: `#ff3d55`
- Purple accent: `#9c5cff`

## Typography

- Interface/display: `Inter, Arial, sans-serif`
- Code/terminal: `JetBrains Mono, Consolas, monospace`
- Metric display: `Orbitron, Arial Black, sans-serif`

## Production Rules

- `VISUAL-SCREENPLAY-V2.md` is the contract.
- Use stills only as optional concept art.
- Build final scenes as layered animated objects: text, panels, counters, rays, meters, source cards, data objects, callouts, and avatar crops.
- Do not use one visual treatment across many unrelated segments.
- Use VS Code/terminal scenes only when the segment actually calls for code, local files, CLI, or scripts.
- Cursor appears only when it clicks, selects, underlines, opens, or drags attention to something meaningful.
- Before full approval, inspect midpoint frames for every segment.
