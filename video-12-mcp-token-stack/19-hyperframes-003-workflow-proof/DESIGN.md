# Segment 003 Workflow Proof Design

Goal: turn the revised Segment 003 narration into a faceless HyperFrames proof that explains on-demand context injection step by step.

Style:

- Use `003C.png` as conceptual guidance, but rebuild as live HyperFrames objects.
- Dense technical editorial look: dark canvas, cyan search energy, green optimized context, gray unrelated library cards.
- Reveal the workflow in sequence. Do not show Step 3 loaded context during Step 1 or Step 2.

Motion plan:

1. **Step 1 - Minimal initial load**: faded tool library appears outside context; only a compact Tool Search card enters the model context window.
2. **Step 2 - Select relevant tools**: a blue search beam scans the library; Jira Task, Sentry Error, and AWS Metrics brighten while unrelated tools remain gray.
3. **Step 3 - Inject matched definitions**: selected cards travel into the green optimized context window; unrelated cards stay outside.
4. **Payoff - 77K to 8.7K**: workflow compresses into token meters and a final "serious improvement" badge.

What not to do:

- Do not use a static 003C background.
- Do not show all three steps fully loaded from the beginning.
- Do not rely on plain fades as the main motion.
- Do not make the final frame a generic dark text card.
