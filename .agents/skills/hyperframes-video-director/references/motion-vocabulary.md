# Motion Vocabulary

Use these as named, reusable patterns while designing Jarvis HyperFrames scenes.

## Assembly

- **definition-stack**: bars or cards enter from alternating sides and lock into a vertical stack.
- **panel-snap-in**: side diagnostic panels arrive from off-screen and settle with a small bounce.
- **context-ingestion**: pages/cards flow along glowing paths into a central context window.
- **layer-build**: capabilities appear one at a time until a complete stack is visible.
- **bench-setup**: charts, axes, labels, and result cards assemble like a test bench.

## Transformation

- **payload-compression**: large result block collapses into a smaller structured summary.
- **schema-slimdown**: large schema shrinks while clean fields remain highlighted.
- **mess-to-order**: scattered cards reorganize into rows, layers, or columns.
- **overload-escalation**: meter/needle moves from green to yellow to red as objects pile up.
- **boundary-crossing**: cards pass through gates or boundary lines to show data movement.

## Still-Image Animation

- **tile-dissolve-out**: image splits into tiles that stagger out.
- **tile-assemble-in**: tiles fly or fade in to build the next image.
- **vertical-swap**: current still exits upward while the next rises from below.
- **slice-wipe**: image is revealed through horizontal or vertical bands.
- **parallax-push**: still background, foreground labels, and highlights move at different depths.
- **spotlight-scan**: a light band or outline scans the important region.
- **annotation-burst**: arrows, circles, callouts, or warnings appear around the still.

## Evidence And Code

- **simulated-ide-focus**: code editor appears with line highlights, cursor movement, and snippets expanding.
- **terminal-proof**: commands and outputs type in, then key lines are boxed or brightened.
- **source-card-zoom**: website/article/screenshot appears, camera pushes into the relevant evidence.
- **diff-reveal**: before/after code or configuration changes are highlighted line by line.

## Transitions

- **panel-push**: one full scene pushes another out horizontally or vertically.
- **card-fan-sweep**: several cards sweep across frame to reveal the next scene.
- **mask-window**: next scene appears inside a growing window, browser frame, or device frame.
- **warning-flash-cut**: a warning frame pulses, then cuts into the error/problem visual.
- **data-stream-wipe**: moving particles or pages become the wipe between scenes.

## Timing Defaults

- Beat entrance: 0.4-0.9s.
- Stagger between repeated objects: 0.05-0.18s.
- Read hold after build: 1-3s.
- Camera push: 6-10s across a full scene, very subtle.
- Avoid using the same transition twice in a row unless repetition is part of the point.
