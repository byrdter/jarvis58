# Codex Sites In Video Production

Sites are a supporting production surface for Jarvis videos. They do **not** replace HyperFrames.

HyperFrames remains the final video composition system for VO timing, motion, captions, transitions, callouts, and render control. Sites create live app-like surfaces that can be used, reviewed, screen-recorded, captured, or embedded as material inside HyperFrames scenes.

## When To Use Sites

Use a Site when an idea is stronger as a working surface than as a static slide:

- dashboard
- simulator
- planner
- review board
- evidence explorer
- comparison tool
- executive memo
- source/provenance hub
- command center
- interactive data story

OpenAI's framing is useful: Sites can turn things that might have been docs or slides into mini apps, internal tools, and shareable resources.

## Three Roles

### 1. Production Tool

Use Sites to help make the video:

- episode command center
- visual treatment board
- scene status tracker
- asset inventory browser
- claim/source review board
- render review workspace
- title/thumbnail decision board

### 2. On-Screen Visual Source

Use Sites as material viewers see in the video:

- AI router simulator
- model decision dashboard
- agent stack map
- transcript intelligence board
- research evidence explorer
- Claude/Codex/Gemini comparison surface
- production command center walkthrough

Capture interactions from the Site, then bring the footage or screenshots into HyperFrames for final timing and polish.

### 3. Companion Asset

Use Sites as resources after the video:

- viewer resource hub
- checklist
- calculator
- demo companion
- source list
- interactive memo
- decision tool

## Workflow

```text
concept
  -> Site prototype
  -> interact / record / screenshot
  -> HyperFrames scene
  -> final video
```

## Capture Patterns

- Type into the Site and show results updating.
- Click filters, tabs, rows, or cards.
- Scroll through a source/evidence board.
- Zoom into a chart, table, or timeline.
- Show a decision changing after a slider or prompt update.
- Record a clean 5-20 second interaction as B-roll.
- Use screenshots as HyperFrames layers for annotations, zooms, and callouts.

## Guardrails

- Do not let Sites become the final video renderer.
- Do not use a Site if a precise VO-synced animation is easier directly in HyperFrames.
- Do not show tiny unreadable full-page UI. Crop, zoom, or redesign the Site for screen capture.
- Do not expose secrets, private customer data, tokens, or unapproved internal details.
- If the Site is shareable, define who can access it and whether it is a viewer companion or only a production artifact.
