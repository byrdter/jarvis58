The pipeline runs in five stages. <break time="1s"/>
Stage one. Transcribe. AssemblyAI extracts word-level timestamps from the avatar audio. Every word. Every pause. Every gap. The pipeline scans for gaps longer than one second. Each gap becomes a hard scene boundary. <break time="1s"/>
Stage two. Plan. Claude reads the script and the transcript and decides what visual goes with each scene. Title cards for transitions. Charts for data. Code blocks for technical bits. B-roll for emotional beats. <break time="1s"/>
Stage three. Generate the visuals. Static images from Nano Banana Pro, DALL-E, or Midjourney — whatever you've wired up. Animated graphics from HyperFrames — title sequences, charts, captions, audio-reactive overlays. Cinematic B-roll from Higgsfield via MCP. <break time="1s"/>
Stage four. Compose. Remotion is React for video. Every scene is a component. The avatar clip, the overlay graphic, the captions, the background — all just JSX. Every frame is deterministic. Frame three hundred always looks the same. No render lottery. <break time="1s"/>
Transitions are code, not clicks. Crossfades, wipes, shader effects — all defined in TypeScript, all reusable across videos. Captions come from the same word-level transcript. Synced to the millisecond. Style them however you want. <break time="1s"/>
Stage five. Render. One command. Remotion renders every frame in parallel using a headless Chrome cluster. Ten-minute video, ten minutes to render. <break time="1s"/>
Then a QA loop. A Claude subagent watches the output, checks for motion defects, transition glitches, audio drift. If it passes, you have a finished video. If it doesn't, the pipeline reports what to fix. <break time="1s"/>
