# Video 25 First Mock Plan

## Episode Shape

Working title: **The 2027 AI Builder Roadmap**

Runtime target: 10-12 minutes.

Audience: non-coders, builders, and business owners.

Frame: preparation, not prediction.

Core promise: by 2027, useful builders will understand the operating stack around models: judgment, tools, memory, workflows, evals, observability, deployment, and product taste.

## Scene List

### 1. Cold Open: Beyond Better Chat

Visual: ChatGPT/OpenAI baseline breaks apart into a layered builder cockpit: Claude Code, Codex, GitHub PRs, terminal logs, and a roadmap spine.

Purpose: establish that chat is only the entry point.

Assets: ChatGPT scroll, Claude Code desktop, Codex desktop, GitHub PR/actions.

### 2. The Roadmap Is A Stack

Visual: six depth layers snap into a vertical system: models, tools, memory, workflows, evals, deployment.

Purpose: give the viewer a mental model before details.

Assets: HyperFrames-built layer map.

### 3. Layer 1: Model Judgment

Visual: decision dashboard where the task moves through lanes: fast, deep reasoning, long context, multimodal, coding, local/private.

Purpose: replace "which model is best?" with "which model fits this job?"

Assets: OpenAI, Claude Code, Gemini, Codex, Hugging Face.

### 4. Layer 2: Tools And APIs

Visual: public docs/web-rolls collapse into tool-call cards and an MCP registry rail.

Purpose: show that agents become useful when they can take action.

Assets: OpenAI platform, Anthropic docs, Google AI docs, MCP Registry.

### 5. Layer 3: Memory And Knowledge

Visual: sanitized JARVIS knowledge cockpit using real database counts: 604 sources, 7,487 segments, 10 topics, 8 channels.

Purpose: make Video 25 uniquely Byrddynasty and show why memory matters.

Assets: JARVIS orb, sanitized mock UI, optional later real capture.

### 6. Layer 4: Workflows, Hooks, And Skills

Visual: workflow control panel where one-off prompts become reusable operating procedures.

Purpose: show the difference between asking once and building repeatable systems.

Assets: Claude Code docs, GitHub issues/actions, LangGraph docs, local mock.

### 7. Layer 5: Evals And Observability

Visual: flight recorder with traces, screenshots, logs, failure tags, and regression checks.

Purpose: show why serious AI systems need review loops.

Assets: LangSmith, Arize Phoenix, Weave, GitHub Actions.

### 8. Layer 6: Deployment And Real Systems

Visual: infrastructure map from local builder workstation to endpoints, queues, workers, storage, monitoring.

Purpose: show how AI work leaves the demo.

Assets: RunPod, Modal, Vercel AI SDK, Cloudflare Workers AI, Supabase Vector.

### 9. What To Learn In Order

Visual: infinite-canvas roadmap with nine checkpoints:

1. Prompting and judgment
2. Git and terminal basics
3. APIs and CLI tools
4. MCP/tool calling
5. Memory and retrieval
6. Workflows and skills
7. Evals and observability
8. Deployment
9. Product taste and visual communication

Purpose: give a practical next-step sequence.

Assets: HyperFrames roadmap animation.

### 10. Close: Build The Stack

Visual: all layers recombine into a completed 2027 builder cockpit, then kinetic CTA.

Purpose: close on action, not hype.

Assets: callbacks from the scene set.

## First Mock Deliverable

The first mock is `visual-mock/index.html`. It is not the final video. It is a fast visual direction board that tests:

- whether the layered-depth style feels right
- whether the public captures are enough
- whether the sanitized JARVIS layer is believable
- which local captures are worth replacing later

## Immediate Next Pass After Review

1. Lock or adjust the visual direction.
2. Choose 2-3 real local captures, if any.
3. Draft the 10-12 minute VO script.
4. Record HeyGen VO-only audio.
5. Build final HyperFrames scenes against the actual audio timings.
6. Run `tools/scene-validator.py` and post-render QC before Terry review.
